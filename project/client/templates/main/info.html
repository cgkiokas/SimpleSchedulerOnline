Simple Cyclic Scheduling Online


How to use:

The app can be used in two ways to generate a cyclic executive schedule for a set of real-time tasks. Firstly, by using the table input as shown below to enter the desired tasks specification. Secondly, by uploading a CSV formatted text file. The CSV format must mimic the input table and start with the field headers from cell 1A.

Fields:
  • Period: Defines the period task.
  • Execution: Defines the worst-case execution time of the task.
  • Deadline: Defines a relative deadline with respect to the period of the task. This field can be used to force the scheduler to schedule a task before a specific point in time.
  • Offset: Defines a relative offset with respect to the period of the task. This field can be also used to force the scheduler to schedule the activation instance of a task after a specific point in time.
  • Allowed Jitter: Defines the maximum allowed jitter that a task can experience, i.e. the tolerated amount of time that a task is allowed to be scheduled after its period or deadline.
  • CPU ID: Defines the CPU ID that the task belongs to. This field can be used allow the scheduler to overlap the execution of two tasks under the assumption that they are executed by different processors.
  • Name: Sets the name of the task both in the generated plot and the generate code.
  • Function: Sets the callable name of the C function of the respective task.
